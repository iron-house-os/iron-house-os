# Build 212 — durable production storage

## Outcome

IHOS can now use a private S3-compatible bucket as the document source of truth while retaining the controlled local-volume option. Object keys are generated by the application, database records store private `s3://` URIs, and files are downloaded to a bounded application cache only when local processing or delivery needs a path.

## Object-storage controls

- `IHOS_STORAGE_BACKEND=s3` selects the provider; local remains the explicit default
- bucket, key prefix, endpoint, region, and cache root are environment-configured
- AWS-compatible credentials come only from the protected runtime environment
- uploads keep the existing extension and 100 MB size controls
- SHA-256 is stored as private object metadata and checked after cache misses
- URIs outside the configured bucket or prefix are rejected
- readiness verifies bucket access without creating a public URL

The provider works with AWS S3 and S3-compatible private endpoints supported by `boto3`. Build 212 does not create a bucket, cloud account, or paid service.

## Scheduled recovery bundles

`scripts/scheduled_backup.sh` wraps the Build 209 database/application-data backup with a non-overlapping file lock and verified retention. `scripts/backup_retention.py` verifies every recognized bundle before deleting anything, always preserves the configured minimum number of newest bundles, and enforces both age and maximum-count limits.

Example systemd service and timer units live in `ops/systemd/`. They are templates only and are not installed or enabled automatically.

## Recovery boundary

For local storage, recovery bundles contain the database and authoritative `/app/data` files. For S3 storage, the bundle contains the database and disposable local cache; the private bucket remains the object source of truth. Operators must enable bucket versioning and independent retention/replication, then retain matching database bundles off-host. The existing release drill still proves database and local-file restoration on every relevant change.
