# Build 90 Live Bid Dry Run Plan

## Dry run objective

Use a real or sample tender package to validate the full IHOS workflow without submitting a bid from IHOS output alone.

## Dry run sequence

1. Create project.
2. Register tender docs.
3. Enter sample takeoff quantities.
4. Save takeoff.
5. Build estimate handoff.
6. Enter pricing assumptions.
7. Save estimate workspace.
8. Generate RFQ drafts.
9. Review project readiness.
10. Export estimate workbook.
11. Compare IHOS output against manual estimator review.

## Pass condition

IHOS must preserve traceability from project to documents, takeoff, estimate, RFQs, and readiness without losing data between page reloads.
