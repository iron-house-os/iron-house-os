# Data Integrity Checklist

Use this when testing IHOS with real tender data.

## Project

- Project ID is stable.
- Project name matches tender documents.
- Municipality and owner are correct.
- Bid due date is correct.

## Documents

- Drawing revisions are current.
- Addenda are registered.
- Superseded files are marked.
- Required RFQ attachments are identifiable.

## Takeoff

- Every quantity has category, unit, quantity, and drawing reference.
- Low-confidence items are reviewed.
- Zero quantities are removed or explained.

## Estimate

- Estimate lines trace back to takeoff, quote, or allowance.
- Subcontract scopes are not double counted.
- Markups and contingency are intentional.

## RFQ

- Supplier category targets match scope.
- Required docs are attached.
- Wrong recipients are removed before sending.
